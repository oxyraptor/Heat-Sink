# Heat Sink Optimizer - Code & Model Explanation

This document provides a detailed breakdown of the **Physics Engine** and **Optimization Logic** implemented in `optimizer.py`.

---

## 1. Physical Model Theory

The system solves for the **Equilibrium Surface Temperature** ($T_s$) of the heat sink where the heat generated by the motor equals the heat dissipated via convection and radiation.

### Heat Balance Equation

$$ Q*{in} = Q*{conv} + Q\_{rad} $$

Where:

- $Q_{in}$: Heat Load (Watts) = $(V \times I) - P_{rated}$
- $Q_{conv}$: Convection Heat Transfer
- $Q_{rad}$: Radiation Heat Transfer

### Convection ($Q_{conv}$)

$$ Q*{conv} = h \cdot A*{eff} \cdot (T*s - T*{amb}) $$

- $h$: Convection Coefficient (W/m²K), dynamic based on airflow.
- $A_{eff}$: Effective Surface Area = $A_{base} + N \cdot A_{fin} \cdot \eta_{fin}$
- $\eta_{fin}$: Fin Efficiency (how well heat travels up the fin).

### Radiation ($Q_{rad}$)

$$ Q*{rad} = \sigma \cdot \epsilon \cdot A*{rad} \cdot (T*s^4 - T*{amb}^4) $$

- $\sigma$: Stefan-Boltzmann Constant ($5.67 \times 10^{-8}$)
- $\epsilon$: Emissivity (0.8 for Anodized Aluminum)
- $A_{rad}$: Surface area of the heat sink "envelope".

## 2. Code Breakdown (`optimizer.py`)

### Imports (Lines 1-6)

Standard scientific libraries:

- `numpy`: For numerical operations.
- `scipy.optimize`: `differential_evolution` (Generic Algorithm) and `root_scalar` (Equation Solver).
- `scipy.special`: Bessel functions `i0`, `i1` for Triangular fin efficiency.

### Class: `AirProperties` (Lines 9-45)

**Purpose**: Calculates dynamic air properties based on temperature. Air becomes less dense and more viscous as it heats up, affecting cooling.

- `get_properties(T_kelvin)`:
  - **Density ($\rho$)**: Ideal Gas Law ($P/RT$).
  - **Conductivity ($k$)**: Polynomial fit for air.
  - **Viscosity ($\mu$)**: Sutherland's Law.
  - **Prandtl Number ($Pr$)**: Ratio of momentum diffusivity to thermal diffusivity ($C_p \mu / k$).
  - **Beta ($\beta$)**: Expansion coefficient ($1/T$) driving natural convection.

### Class Hierarchy: `FinGeometry` (Lines 48-164)

**Purpose**: Object-Oriented definition of different fin shapes (Rectangular, Triangular, Trapezoidal).

- `FinGeometry` (Abstract Base):
  - `__init__`: Calculates **Fin Spacing ($s$)** based on Width, N (count), and t_base.
  - `is_valid()`: Checks if spacing > 0.5mm (Manufacturing limit).
- `RectangularFin` (Lines 86-103):
  - `fin_efficiency`: Uses standard `tanh(mLc)/mLc` formula.
  - `surface_area`: $2HL + tL$.
- `TriangularFin` (Lines 105-126):
  - `fin_efficiency`: Uses Bessel functions (`i1/i0`) because cross-section area changes with height.
  - `surface_area`: Uses slant height $\sqrt{H^2 + (t/2)^2}$.
- `TrapezoidalFin` (Lines 128-164):
  - Hybrid model with a `t_tip` (tip thickness) = 50% of base.

### Class: `AdvancedThermalModel` (Lines 168-285)

**Purpose**: The Physics Engine. solving the heat balance.

- `get_heat_load` (Lines 178-192):
  - Calculates Heat Loss. Priority: Efficiency -> Voltage/Current -> Default.
- `solve_equilibrium` (Lines 194-285): **The Core Solver**
  - **Inputs**: Geometry object, Material Conductivity ($k$), Base Plate Thickness.
  - **Logic**:
    1.  Define a `heat_balance_residual(T_s)` function that returns $Q_{out} - Q_{in}$ for a guess temperature $T_s$.
    2.  Inside residual:
        - Get Air Properties at Film Temp $T_{film} = (T_s + T_{amb})/2$.
        - **Calculate $h$ (Convection)**:
          - _Natural_: Uses **Elenbaas/Churchill-Chu** correlations using Rayleigh Number ($Ra_s$).
          - _Forced_: Uses **Teertstra** correlation (or Flat Plate approximation) using Reynolds Number ($Re_L$).
        - **Calculate $Q_{rad}$**: Using Stefan-Boltzmann law.
        - **Calculate $Q_{conv}$**: Using $h$, Surface Area, and Fin Efficiency.
    3.  Use `root_scalar` to find the exact $T_s$ where Residual = 0.
    4.  Return final `T_surface`, `R_total`, `h_avg`.

### Class: `DesignOptimizer` (Lines 289-417)

**Purpose**: Finds the best geometry using Differential Evolution.

- `objective_function` (Lines 347-365):
  - The "Score" to minimize.
  - **Constraints**: If Height > Max or Thickness < Min, return Huge Score ($10^6$).
  - **Evaluation**: Run `model.solve_equilibrium`.
  - **Cost**: `Mass * 10 + Penalty`.
  - **Penalty**: If Temp > MaxTemp, add $(T - T_{max})^2 \times 100$. This "Soft Constraint" guides the optimizer toward valid solutions smoothly.
- `calculate_sensitivity` (Lines 367-386):
  - Calculates "Gradients": How much Temperature changes if we change parameters (N, H, t) by 1%.
  - Used to tell the user: "Fin Height is the most critical factor".

- `optimize` (Lines 388-417):
  - Runs `scipy.differential_evolution` for each geometry type (Rect, Tri, Trap).
  - Picks the best valid design (Lowest Mass that satisfies Temp).
  - Prioritizes Validity: If Temp > Max, Score is high. If valid, Score is Mass.

## 3. Key Algorithms

### Differential Evolution (DE)

Used because the thermal non-linearity (Nu correlations, Radiation) makes the landscape complex and non-differentiable. DE evolves a population of designs to find the global minimum mass.

### Root Finding (Brent's Method)

Used in `solve_equilibrium`. We don't just guess Temperature; we mathematically solve for the exact steady-state temperature where Energy In = Energy Out.
